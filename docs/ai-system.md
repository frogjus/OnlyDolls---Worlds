# StoryForge AI System Design

## 1. Overview

### Core Principle: "Analysis-first, generation-optional"

StoryForge's AI layer exists to **analyze, structure, and validate** existing story material. It does not generate content by default. The AI examines narrative text and extracts structured data -- characters, relationships, events, themes, emotional arcs, pacing metrics, narrative codes -- and organizes them into a living knowledge graph that writers can explore visually.

Generation capabilities (beat suggestions, script drafts, synopsis expansion) exist as an **opt-in assist layer** behind the AI Wand feature. Generation never auto-triggers, never auto-commits, and is always presented as a suggestion that the user explicitly accepts, edits, or dismisses. The Wand is disabled until the user has filled in the synopsis in the Story Sidebar, ensuring human creative input comes first.

### Architectural Boundaries

- All AI calls are abstracted behind `src/lib/ai/` -- no component or API route ever calls the Claude API directly.
- The Claude API is the primary AI provider for text analysis, entity extraction, consistency checking, voice analysis, pacing computation, emotional arc charting, narrative code tagging, and opt-in generation.
- Whisper (OpenAI) or Deepgram handles audio transcription. FFmpeg handles video processing. These feed their output into the Claude-powered text analysis pipeline.
- Voyage AI or local models generate vector embeddings for semantic search via pgvector. Embeddings are not generated by Claude.
- SuperMemory SDK provides the persistent knowledge graph layer with semantic search, contradiction detection, and typed relationship edges.
- All AI operations run asynchronously through BullMQ job queues -- they never block the HTTP request thread.

### Trust Model

Every AI-extracted entity starts in a `proposed` state. Nothing enters the canonical knowledge graph without user confirmation. Confidence scores are displayed for every extraction. Hard factual contradictions (alive/dead, location paradoxes) are flagged automatically; soft interpretive contradictions (motivational, thematic) are surfaced as suggestions requiring human judgment. The system never presents AI uncertainty as certainty.

## 3. Analysis Capabilities

### 3.1 Entity Extraction

**Purpose**: Parse narrative text and extract structured entities with types, attributes, confidence scores, and references back to source positions.

**Pipeline**:
1. Text is chunked (see Section 5.2 for chunking strategy).
2. Each chunk is sent to Claude with surrounding context (previous chunk overlap + world context).
3. Claude returns structured JSON matching the extraction schema.
4. Cross-chunk entity resolution merges duplicates (fuzzy name matching + attribute comparison).
5. Merged entities are persisted in `proposed` state awaiting user confirmation.

**Entity Types**: Character, Location, Event, Relationship, Theme, Object, Faction

**TypeScript Interfaces**:

```typescript
// Input to the extraction pipeline
interface EntityExtractionInput {
  text: string;                          // The text chunk to analyze
  chunkIndex: number;                    // Position in the overall document
  totalChunks: number;                   // Total number of chunks
  previousContext: string;               // Last 500 tokens of previous chunk
  worldContext: WorldContext;             // Existing entities, world rules, etc.
  sourceId: string;                      // Reference to SourceMaterial record
  sourceOffsetStart: number;             // Character offset in original document
  sourceOffsetEnd: number;               // Character offset end
}

// A single extracted entity
interface ExtractedEntity {
  type: EntityType;                      // 'character' | 'location' | 'event' | 'relationship' | 'theme' | 'object' | 'faction'
  name: string;                          // Primary name or label
  aliases: string[];                     // Alternative names, nicknames, titles
  attributes: Record<string, unknown>;   // Type-specific attributes (see below)
  confidence: number;                    // 0-1 overall confidence
  attributeConfidence: Record<string, number>; // Per-attribute confidence
  sourceReferences: SourceReference[];   // Where in the text this was found
  proposedRelationships: ProposedRelationship[]; // Relationships detected in context
}

// Character-specific attributes
interface CharacterAttributes {
  physicalDescription?: string;
  age?: string;                          // May be approximate: "early 30s"
  gender?: string;
  role?: string;                         // Protagonist, antagonist, supporting, etc.
  occupation?: string;
  traits: string[];                      // Personality traits observed
  status: 'alive' | 'dead' | 'unknown';
  firstAppearance: SourceReference;
}

// Location-specific attributes
interface LocationAttributes {
  description?: string;
  type?: string;                         // City, room, planet, forest, etc.
  parentLocation?: string;               // Containing location name
  significance?: string;                 // Why it matters to the story
}

// Event-specific attributes
interface EventAttributes {
  eventType: 'action' | 'dialogue' | 'discovery' | 'transformation' | 'revelation' | 'decision';
  participants: string[];                // Character names involved
  location?: string;                     // Where it happens
  chronologicalOrder?: number;           // Estimated position in fabula
  narrativeOrder: number;                // Position in sjuzhet (derived from chunk position)
  significance: 'major' | 'moderate' | 'minor';
}

// Source reference linking extraction to original text
interface SourceReference {
  sourceId: string;
  offsetStart: number;
  offsetEnd: number;
  quotedText: string;                    // The exact text that triggered extraction
}

// Output from a single chunk extraction
interface EntityExtractionOutput {
  entities: ExtractedEntity[];
  unresolvedReferences: string[];        // Names mentioned but not enough info to type
  chunkSummary: string;                  // Brief summary of chunk content for context passing
}
```

**Cross-Chunk Entity Resolution**:

```typescript
// src/lib/ai/analysis/extract-entities.ts

interface EntityResolutionConfig {
  nameMatchThreshold: number;       // 0.85 — fuzzy match threshold for names
  attributeOverlapThreshold: number; // 0.6 — minimum attribute overlap to consider merge
  requireUserConfirmation: boolean;  // true for ambiguous merges
}

// Resolution algorithm:
// 1. Collect all entities across all chunks.
// 2. Group by type.
// 3. Within each type, compute pairwise name similarity (Jaro-Winkler).
// 4. For pairs above nameMatchThreshold, compare attributes.
// 5. If attribute overlap is above attributeOverlapThreshold, auto-merge (keep highest confidence per attribute).
// 6. If name matches but attributes conflict, flag for user review.
// 7. Merge alias lists across matched entities.
// 8. Assign merged entity a new aggregate confidence = max(individual confidences).
```

### 3.2 Relationship Mapping

**Purpose**: Infer typed, weighted, temporal relationships between entities extracted from a scene or chapter.

**TypeScript Interfaces**:

```typescript
interface RelationshipMappingInput {
  entities: ExtractedEntity[];           // Entities in scope (scene or chapter)
  sceneText: string;                     // The source text
  existingRelationships: Relationship[]; // Already-known relationships in the world
  worldContext: WorldContext;
}

interface MappedRelationship {
  sourceEntity: string;                  // Name or ID
  targetEntity: string;                  // Name or ID
  type: RelationshipType;
  subtype?: string;                      // More specific: 'parent', 'rival', 'employer'
  weight: number;                        // 0-1 strength/significance
  sentiment: 'positive' | 'negative' | 'neutral' | 'ambivalent';
  bidirectional: boolean;                // true = symmetric (friends), false = asymmetric (mentor/mentee)
  inferredInverse?: {                    // If not bidirectional, the inverse relationship
    type: RelationshipType;
    subtype?: string;
  };
  validFrom?: string;                    // Scene/event ID where relationship begins
  validTo?: string;                      // Scene/event ID where relationship ends (if it does)
  evidence: SourceReference[];           // Text evidence
  confidence: number;                    // 0-1
  isNew: boolean;                        // true = not in existingRelationships
  changesExisting?: string;              // ID of existing relationship this modifies
}

type RelationshipType =
  | 'familial'
  | 'romantic'
  | 'professional'
  | 'antagonistic'
  | 'mentorship'
  | 'alliance'
  | 'friendship'
  | 'rivalry'
  | 'political'
  | 'supernatural'
  | 'causal'
  | 'dependency'
  | 'other';

interface RelationshipMappingOutput {
  relationships: MappedRelationship[];
  relationshipChanges: RelationshipChange[];  // Existing relationships that evolved
  potentialConflicts: string[];               // Notes on relationships that seem contradictory
}

interface RelationshipChange {
  existingRelationshipId: string;
  changeType: 'strengthened' | 'weakened' | 'transformed' | 'ended';
  newType?: RelationshipType;
  newWeight?: number;
  newSentiment?: string;
  evidence: SourceReference[];
  confidence: number;
}
```

### 3.3 Consistency and Contradiction Checking

**Purpose**: Detect contradictions between new content and the established world state. Uses a tiered approach: hard contradictions are flagged automatically with high confidence; soft contradictions are suggestions requiring human judgment.

**Tiered Approach**:

| Tier | Type | Examples | Confidence | Action |
|------|------|----------|------------|--------|
| **Hard** | Factual/temporal | Dead character appears alive; character in two locations simultaneously; timeline paradox; age inconsistency | 0.9-1.0 | Auto-flagged, high severity |
| **Medium** | Structural/rule-based | Violation of defined world rules (magic costs not paid); established capability contradicted | 0.6-0.8 | Auto-flagged, medium severity |
| **Soft** | Behavioral/thematic | Character acts against established psychology; tone inconsistency; motivation contradiction | 0.3-0.6 | Suggestion, low severity, requires human judgment |

**TypeScript Interfaces**:

```typescript
interface ConsistencyCheckInput {
  newContent: string;                     // New text being checked
  newEntities: ExtractedEntity[];         // Entities extracted from new content
  worldState: WorldState;                 // Current canonical world state
  worldRules: WorldRule[];                // Defined rules/constraints
  scope: 'full' | 'character' | 'timeline' | 'location' | 'rules';
}

interface WorldState {
  characters: CharacterState[];           // Current state of all characters
  locations: LocationState[];
  events: EventRecord[];                  // All known events in chronological order
  relationships: Relationship[];
  factions: FactionState[];
  activeArcs: ArcState[];
  worldRules: WorldRule[];
}

interface CharacterState {
  id: string;
  name: string;
  status: 'alive' | 'dead' | 'unknown';
  lastKnownLocation?: string;
  currentAge?: string;
  knownTraits: string[];
  knownRelationships: string[];           // Summary strings
  lastSeenEvent?: string;                 // Event ID
}

interface Contradiction {
  id: string;
  tier: 'hard' | 'medium' | 'soft';
  severity: 'critical' | 'warning' | 'suggestion';
  confidence: number;                     // 0-1
  category: ContradictionCategory;
  description: string;                    // Human-readable explanation
  existingFact: {
    statement: string;
    source: SourceReference;
  };
  contradictingFact: {
    statement: string;
    source: SourceReference;
  };
  affectedEntities: string[];             // Entity IDs
  suggestedResolution?: string;           // How to fix it (optional)
}

type ContradictionCategory =
  | 'character_status'        // alive/dead mismatch
  | 'timeline_paradox'        // event ordering impossible
  | 'location_conflict'       // character in two places at once
  | 'age_inconsistency'       // age does not match timeline
  | 'attribute_change'        // unexplained physical/trait change
  | 'world_rule_violation'    // breaks defined world rules
  | 'relationship_conflict'   // contradicts established relationship
  | 'behavioral'              // acts against established character
  | 'motivational'            // motivation contradicts backstory
  | 'tonal'                   // tone/voice inconsistency
  | 'continuity_error'        // object/detail mismatch
  | 'setup_violation';        // foreshadowing promise broken

interface ConsistencyCheckOutput {
  contradictions: Contradiction[];
  warnings: Contradiction[];              // Medium tier
  suggestions: Contradiction[];           // Soft tier
  consistencyScore: number;               // 0-1 overall
  checkedAspects: string[];               // What was checked
}
```

**Prompt Template** (abbreviated):

```typescript
// src/lib/ai/prompts/consistency-check.ts

OUTPUT FORMAT: Valid JSON matching ConsistencyCheckOutput schema.`,

    user: `WORLD STATE:
${formatWorldState(input.worldState)}

${formatWorldRules(input.worldRules)}

NEW CONTENT TO CHECK:
---
${input.newContent}
---

ENTITIES EXTRACTED FROM NEW CONTENT:
${JSON.stringify(input.newEntities, null, 2)}

Check this new content for contradictions against the world state. Return valid JSON.`
  };
}
```

### 3.4 Character Voice Analysis

**Purpose**: Build a quantitative voice profile for each character based on their dialogue, and detect when characters sound identical or deviate from their established patterns.

**TypeScript Interfaces**:

```typescript
interface VoiceAnalysisInput {
  characterName: string;
  dialogueLines: DialogueLine[];         // All dialogue for this character
  existingProfile?: VoiceProfile;        // Profile to compare against (if updating)
  otherProfiles?: VoiceProfile[];        // Other characters' profiles for cross-comparison
}

interface DialogueLine {
  text: string;
  sceneId: string;
  emotionalContext?: string;             // Scene emotional context
  speakingTo?: string;                   // Who they are addressing
  sourceReference: SourceReference;
}

interface VoiceProfile {
  characterId: string;
  characterName: string;

  // Vocabulary metrics
  vocabularyLevel: 'simple' | 'moderate' | 'advanced' | 'literary';
  averageWordLength: number;
  uniqueWordRatio: number;               // Type-token ratio
  dominantRegister: 'formal' | 'informal' | 'colloquial' | 'archaic' | 'technical';
  jargonDomains: string[];               // e.g., ['military', 'medical', 'legal']

  // Sentence structure
  averageSentenceLength: number;         // In words
  sentenceLengthVariance: number;
  preferredStructures: string[];         // e.g., ['declarative', 'interrogative', 'imperative']
  fragmentUsage: 'none' | 'rare' | 'moderate' | 'frequent';

  // Speech patterns
  speechTics: string[];                  // Repeated phrases, filler words, catchphrases
  oathsExpletives: string[];             // Characteristic exclamations
  addressPatterns: string[];             // How they address others: "sir", "dude", first names
  contractionUsage: 'always' | 'often' | 'sometimes' | 'never';

  // Rhetorical tendencies
  usesMetaphor: boolean;
  usesIrony: boolean;
  asksQuestions: 'rarely' | 'sometimes' | 'frequently';
  interrupts: boolean;
  monologueTendency: 'terse' | 'moderate' | 'verbose';

  // Emotional expression
  emotionalRange: 'restrained' | 'moderate' | 'expressive';
  defaultTone: string;                   // e.g., 'sardonic', 'earnest', 'guarded'

  // Statistical
  totalLinesAnalyzed: number;
  confidence: number;                    // 0-1
}

interface VoiceAnalysisOutput {
  profile: VoiceProfile;
  deviations: VoiceDeviation[];          // Lines that deviate from established profile
  crossCharacterSimilarity: CrossCharacterMatch[];
}

interface VoiceDeviation {
  dialogueLine: DialogueLine;
  deviationType: string;                 // What aspect deviated
  expected: string;                      // What was expected based on profile
  actual: string;                        // What was found
  severity: 'minor' | 'notable' | 'significant';
  possibleExplanation?: string;          // Stress, deception, character growth, etc.
}

interface CrossCharacterMatch {
  otherCharacterName: string;
  similarityScore: number;               // 0-1, higher = more similar (bad)
  sharedPatterns: string[];              // What they have in common
  recommendation: string;                // How to differentiate them
}
```

### 3.5 Pacing and Rhythm Analysis

**Purpose**: Compute quantitative pacing metrics for each scene, chapter, and act. Detect pacing problems such as lulls, unsustained tension, and monotonous rhythm. Compare actual pacing against expected curves from story structure templates.

**TypeScript Interfaces**:

```typescript
interface PacingAnalysisInput {
  text: string;                          // Scene or chapter text
  sceneId: string;
  scenePosition: number;                 // Percentage position in overall narrative (0-100)
  structureTemplate?: {                  // Active structure template (if any)
    name: string;                        // e.g., 'Save the Cat'
    expectedBeat?: string;               // Expected beat at this position
    expectedTensionLevel?: number;       // Expected tension 0-1 at this % position
  };
  previousScenePacing?: PacingMetric;    // For trend analysis
  nextScenePacing?: PacingMetric;        // For trend analysis (if available)
}

interface PacingMetric {
  sceneId: string;
  scenePosition: number;                 // 0-100 percentage

  // Raw metrics (0.0-1.0 normalized)
  actionDensity: number;                 // Proportion of text that is physical action
  dialogueRatio: number;                 // Proportion of text that is dialogue
  descriptionDensity: number;            // Proportion of text that is description/exposition
  introspectionDensity: number;          // Proportion that is internal thought

  // Derived metrics
  tensionLevel: number;                  // 0-1 composite tension assessment
  emotionalIntensity: number;            // 0-1 emotional charge of the scene
  informationDensity: number;            // 0-1 how much new info is conveyed
  sceneLength: number;                   // Word count

  // Rhythm indicators
  sentenceLengthVariance: number;        // Higher = more rhythmic variety
  paragraphLengthVariance: number;
  dialogueToNarrativeRatio: number;      // Balance of dialogue vs. narrative

  // Structural
  netValueChange: number;                // -1 to +1, McKee value shift
  valueChangeDirection: 'positive' | 'negative' | 'neutral' | 'mixed';

  confidence: number;
}

interface PacingAnalysisOutput {
  metric: PacingMetric;
  problems: PacingProblem[];
  templateDeviation?: {                  // How far from expected pacing curve
    expectedTension: number;
    actualTension: number;
    deviation: number;                   // Absolute difference
    interpretation: string;              // 'ahead of expected rhythm' / 'behind' / 'on track'
  };
}

interface PacingProblem {
  type: 'lull' | 'unsustained_tension' | 'monotonous_rhythm' | 'abrupt_shift'
      | 'exposition_dump' | 'dialogue_heavy' | 'action_fatigue';
  severity: 'minor' | 'moderate' | 'significant';
  description: string;
  suggestion?: string;
}
```

```typescript
- emotionalIntensity: emotional charge/weight

- sceneLength: word count (raw number)

- "unsustained_tension": tension peaks but cannot maintain for expected duration

- "exposition_dump": information density spikes unnaturally
- "dialogue_heavy": dialogue ratio exceeds 0.8 for extended sections
```

### 5.3 Confidence Scoring System

```typescript
// src/lib/ai/utils/confidence.ts

interface ConfidenceThresholds {
  autoAccept: number;          // >= this: entity/result accepted without review
  reviewRequired: number;      // >= this but < autoAccept: shown for user review
  lowConfidence: number;       // >= this but < reviewRequired: flagged as uncertain
  discard: number;             // < this: not shown to user, logged only
}

const DEFAULT_THRESHOLDS: ConfidenceThresholds = {
  autoAccept: 0.95,            // Near-certain: "John is Mary's father" (explicitly stated)
  reviewRequired: 0.70,        // Probable: strong textual evidence but not explicit
  lowConfidence: 0.40,         // Possible: inferred from context, may be wrong
  discard: 0.20,               // Too uncertain to present
};

// Per-task threshold overrides
const TASK_THRESHOLDS: Record<TaskType, Partial<ConfidenceThresholds>> = {
  entityExtraction: {
    autoAccept: 0.95,
    reviewRequired: 0.65,      // Lower bar because user confirmation is always required
  },
  consistencyCheck: {
    autoAccept: 0.90,          // Hard contradictions auto-flag at high confidence
    reviewRequired: 0.50,      // Soft contradictions shown even at moderate confidence
    lowConfidence: 0.30,       // Interpretive suggestions have low bar
  },
  voiceAnalysis: {
    autoAccept: 0.90,
    reviewRequired: 0.60,
  },
};

// Confidence display strategy
interface ConfidenceDisplay {
  level: 'high' | 'medium' | 'low';
  color: string;                // For UI: green, yellow, orange
  showScore: boolean;           // Whether to show the numeric score
  requiresAction: boolean;     // Whether user must confirm/dismiss
  label: string;               // Human-readable label
}

function getConfidenceDisplay(
  score: number,
  taskType: TaskType
): ConfidenceDisplay {
  const thresholds = {
    ...DEFAULT_THRESHOLDS,
    ...TASK_THRESHOLDS[taskType],
  };

  if (score >= thresholds.autoAccept) {
    return {
      level: 'high',
      color: 'green',
      showScore: false,
      requiresAction: false,
      label: 'High confidence',
    };
  }
  if (score >= thresholds.reviewRequired) {
    return {
      level: 'medium',
      color: 'yellow',
      showScore: true,
      requiresAction: true,
      label: 'Review suggested',
    };
  }
  if (score >= thresholds.lowConfidence) {
    return {
      level: 'low',
      color: 'orange',
      showScore: true,
      requiresAction: true,
      label: 'Low confidence - verify',
    };
  }
  // Below lowConfidence threshold: should have been discarded
  return {
    level: 'low',
    color: 'red',
    showScore: true,
    requiresAction: true,
    label: 'Very uncertain',
  };
}
```

```typescript
  private activeRequests: number = 0;
  private requestQueue: Array<() => void> = [];
  private requestTimestamps: number[] = [];

  constructor(config?: Partial<RateLimitConfig>) {
    this.client = new Anthropic({
      apiKey: process.env.ANTHROPIC_API_KEY,
    });
    // ...rate limit initialization
  }

  async request(params: Anthropic.MessageCreateParams): Promise<Anthropic.Message> {
    await this.acquireSlot();
    try {
      return await this.executeWithRetry(params);
    } finally {
      this.releaseSlot();
    }
  }

  private async executeWithRetry(
    params: Anthropic.MessageCreateParams,
    attempt: number = 0
  ): Promise<Anthropic.Message> {
    try {
      return await this.client.messages.create(params);
    } catch (error) {
      if (this.isRetryableError(error) && attempt < this.config.maxRetries) {
        const backoff = Math.min(
          this.config.backoffBaseMs * Math.pow(2, attempt),
          this.config.backoffMaxMs
        );
        await this.sleep(backoff);
        return this.executeWithRetry(params, attempt + 1);
      }
      throw error;
    }
  }

  private isRetryableError(error: unknown): boolean {
    if (error instanceof Anthropic.RateLimitError) return true;
    if (error instanceof Anthropic.InternalServerError) return true;
    if (error instanceof Anthropic.APIConnectionError) return true;
    return false;
  }

  // ...slot management, rate tracking
}
```

### 6.2 Response Parsing and Validation

```typescript
// All Claude responses are expected as JSON. Parsing handles common failure modes.

interface ParseResult<T> {
  success: boolean;
  data?: T;
  error?: string;
  rawResponse?: string;
}

async function parseAIResponse<T>(
  response: Anthropic.Message,
  schema: z.ZodSchema<T>     // Zod schema for validation
): Promise<ParseResult<T>> {
  const text = response.content
    .filter((block): block is Anthropic.TextBlock => block.type === 'text')
    .map(block => block.text)
    .join('');

  // Step 1: Try direct JSON parse
  try {
    const parsed = JSON.parse(text);
    const validated = schema.parse(parsed);
    return { success: true, data: validated };
  } catch {
    // Continue to fallback strategies
  }

  // Step 2: Try extracting JSON from markdown code blocks
  const jsonMatch = text.match(/```(?:json)?\s*([\s\S]*?)```/);
  if (jsonMatch) {
    try {
      const parsed = JSON.parse(jsonMatch[1]);
      const validated = schema.parse(parsed);
      return { success: true, data: validated };
    } catch {
      // Continue
    }
  }

  // Step 3: Try extracting JSON object/array from mixed text
  const objectMatch = text.match(/\{[\s\S]*\}/);
  if (objectMatch) {
    try {
      const parsed = JSON.parse(objectMatch[0]);
      const validated = schema.parse(parsed);
      return { success: true, data: validated };
    } catch {
      // Fall through to error
    }
  }

  return {
    success: false,
    error: 'Failed to parse valid JSON from AI response',
    rawResponse: text,
  };
}
```

```typescript
    public details?: Record<string, unknown>
  ) {
    super(message);
    this.name = 'AIError';
  }
}

type AIErrorCode =
  | 'RATE_LIMITED'           // API rate limit hit
  | 'TOKEN_LIMIT_EXCEEDED'  // Input too long for context window
  | 'MALFORMED_RESPONSE'    // Claude returned unparseable output
  | 'VALIDATION_FAILED'     // Response did not match expected schema
  | 'API_ERROR'             // Generic API error
  | 'CONNECTION_ERROR'      // Network issue
  | 'TIMEOUT'               // Request took too long
  | 'QUOTA_EXCEEDED'        // Monthly/daily usage limit
  | 'INVALID_INPUT'         // Input validation failed before API call
  | 'CONTEXT_TOO_LARGE';    // World context exceeds budget after truncation

// Error handler at the API route level
function handleAIError(error: unknown): { error: string; code: string; status: number } {
  if (error instanceof AIError) {
    switch (error.code) {
      case 'RATE_LIMITED':
        return { error: 'Analysis service is busy. Please try again shortly.', code: 'RATE_LIMITED', status: 429 };
      case 'TOKEN_LIMIT_EXCEEDED':
        return { error: 'Text is too long for single analysis. It will be processed in chunks.', code: 'TOKEN_LIMIT', status: 413 };
      case 'MALFORMED_RESPONSE':
        return { error: 'Analysis produced unexpected results. Retrying with simplified analysis.', code: 'PARSE_ERROR', status: 502 };
      case 'QUOTA_EXCEEDED':
        return { error: 'Analysis quota reached for this period.', code: 'QUOTA_EXCEEDED', status: 402 };
      default:
        return { error: 'An error occurred during analysis.', code: 'AI_ERROR', status: 500 };
    }
  }
  return { error: 'An unexpected error occurred.', code: 'UNKNOWN', status: 500 };
}
```

### 7.3 Async Processing via BullMQ

All AI operations run through BullMQ job queues. This ensures the HTTP request thread is never blocked by an API call that may take 5-30 seconds.

```typescript

import Redis from 'ioredis';

const ingestionQueue = new Queue('ai-ingestion', { connection: redis });

// - Generation (wand): high priority, user is waiting
// - Analysis (entity extraction during ingestion): normal priority
// - Batch analysis (pacing, emotional arcs): low priority, background processing
// - Consistency checks: normal priority when triggered by edit, low when periodic

// Job types

  type: 'entity_extraction' | 'relationship_mapping' | 'consistency_check'
      | 'voice_analysis' | 'pacing_analysis' | 'emotional_arc' | 'narrative_coding'
      | 'beat_generation' | 'script_generation' | 'synopsis_expansion';
  worldId: string;
  input: unknown;                        // Task-specific input
  priority: 'high' | 'normal' | 'low';
  userId: string;                        // For cost tracking
  callbackChannel?: string;              // WebSocket channel to notify on completion
}

// Worker configuration
const analysisWorker = new Worker('ai-analysis', async (job) => {
  const { type, input, worldId } = job.data as AIJob;

  // Check cache first
  const cached = await checkCache(type, input, worldId);
  if (cached) return cached;

  // Execute analysis
  const result = await executeAnalysis(type, input);

  // Cache result
  await cacheResult(type, input, worldId, result);

  // Notify via WebSocket if callback channel specified
  if (job.data.callbackChannel) {
    await notifyClient(job.data.callbackChannel, { type, result });
  }

  return result;
}, {
  connection: redis,
  concurrency: 3,                        // Max 3 concurrent AI calls per worker
  limiter: {
    max: 50,                             // Max 50 jobs per minute
    duration: 60000,
  },
});
```

```typescript
// tests/regression/ai/extraction-quality.test.ts

// Regression tests track extraction quality across prompt changes.
// Each test uses a fixed input text and asserts on expected extractions.
// When prompts are updated, these tests catch quality regressions.

import { describe, it, expect } from 'vitest';
import { extractEntities } from '@/lib/ai/analysis/extract-entities';
import { REGRESSION_FIXTURES } from '@/tests/fixtures/extraction-regression';

describe('Entity extraction regression', () => {
  // These tests run against the actual Claude API in CI (with a test API key)
  // or against recorded responses in local development.

  for (const fixture of REGRESSION_FIXTURES) {
    it(`${fixture.name}: extracts expected entities`, async () => {
      const result = await extractEntities(fixture.input);

      for (const expected of fixture.expectedEntities) {
        const found = result.entities.find(
          e => e.type === expected.type && e.name === expected.name
        );
        expect(found, `Expected to find ${expected.type} "${expected.name}"`).toBeDefined();

        if (expected.minConfidence) {
          expect(found!.confidence).toBeGreaterThanOrEqual(expected.minConfidence);
        }

        for (const [attr, value] of Object.entries(expected.attributes || {})) {
          expect(found!.attributes[attr]).toBe(value);
        }
      }
    });

    it(`${fixture.name}: does not produce false positives`, async () => {
      const result = await extractEntities(fixture.input);

      for (const unexpected of fixture.unexpectedEntities || []) {
        const found = result.entities.find(
          e => e.type === unexpected.type && e.name === unexpected.name
        );
        expect(found, `Should NOT find ${unexpected.type} "${unexpected.name}"`).toBeUndefined();
      }
    });
  }
```
